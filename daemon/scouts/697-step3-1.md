## Pre-merge review checklist

### Cache-key invariants (umbrella body, ratified v155)

- [ ] Cache key for `/api/inbox/[address]` GET excludes auth'd branches OR includes a verified-address-hash suffix (so no public cache HIT serves auth'd content)
- [ ] Auth'd branches set `Cache-Control: private, no-store`
- [ ] BIP-322 / SIP-018 gate runs **before** any cache HIT can be served (the agent-news#802 bug class)

### Read-source flip (the load-bearing change)

- [ ] Response `unreadCount` field sourced from D1 `SELECT COUNT(*) FROM inbox_messages WHERE to_btc_address=? AND is_reply=0 AND read_at IS NULL` (not from KV's cached aggregate counter)
- [ ] KV's `unreadCount` field on `inbox:agent:<addr>` index is no longer read on this path (and ideally no longer written, per Step 4 deferred — but at minimum: not a hard dependency for this PR)
- [ ] D1 query has explicit index on `(to_btc_address, is_reply, read_at)` or equivalent — verify migration includes it; check `EXPLAIN QUERY PLAN` doesn't show full scan

### Test coverage (per v137 + v144)

- [ ] **Positive-path test (v144):** registered agent + 2 unread + 0 read → response.unreadCount === 2 (asserts the D1 count is surfaced correctly)
- [ ] **Cache-key auth-branch invariant test (v137 — agent-news#802 bug class):** synthetic cache HIT pre-auth-gate must NOT serve cached body; assert via mock cache layer
- [ ] **Empty-inbox edge:** registered agent + 0 messages → response.unreadCount === 0, totalCount === 0, drift === 0
- [ ] **Power-user edge:** registered agent + N>100 messages, all unread → response.unreadCount === N (no overflow / pagination interaction with COUNT(*))
- [ ] **`is_reply=0` filter test:** registered agent + mixed messages and replies → only non-reply unreads counted
